## üìä What Each Component Does

### ETL Pipeline (`scripts/fetch-daily.js`)
**What**: Fetches stock data from Finnhub API  
**When**: Run with `npm run fetch:daily`  
**Output**: Hot cache + raw backups + manifest  
**Config**: 7 environment variables  
‚Üí See: **[ETL_FORENSICS_SUMMARY.md](ETL_FORENSICS_SUMMARY.md)** ‚Üí Section I

### Hot Cache System (`public/data/hot/`)
**What**: Stores fresh stock data for app runtime  
**When**: Generated by ETL, loaded by app on startup  
**Format**: Static JSON files  
**Fallback**: Uses mock data if unavailable  
‚Üí See: **[ETL_FORENSICS_SUMMARY.md](ETL_FORENSICS_SUMMARY.md)** ‚Üí Section II

### Forensic Analysis (`src/components/ForensicsAnalysis.jsx`)
**What**: Real-time risk scoring for stocks  
**When**: User navigates to Forensics view  
**Output**: Risk score + red flags + recommendation  
**Data Source**: Finnhub API or hot cache  
‚Üí See: **[ETL_FORENSICS_SUMMARY.md](ETL_FORENSICS_SUMMARY.md)** ‚Üí Section III

### App Integration (`src/App.jsx`)
**What**: Main application logic  
**Features**: Search, results, deep dive, forensics  
**Mode**: Mock mode (no live API required)  
**Status**: ‚úÖ Production ready  
‚Üí See: **[ETL_FORENSICS_SUMMARY.md](ETL_FORENSICS_SUMMARY.md)** ‚Üí Section IV

## üìû Quick Reference

### Environment Variables (ETL)
```
INGEST_SYMBOLS_FILE        Custom symbol list path
INGEST_CONCURRENCY         Parallel workers (default: 4)
INGEST_RETRIES             Retry attempts (default: 2)
ALLOW_INSECURE_SSL         Disable TLS (default: false)
```

### File Locations
```
scripts/fetch-daily.js              ETL pipeline
public/data/hot/hotCache.json       Generated cache
public/data/manifest.json           Metadata & errors
public/data/raw/2026-01-22/         Per-symbol backups
symbols/tech_stocks.json            Example custom list
```

### Key Commands
| What | Command |
|------|---------|
| Start app | `npm run dev` |
| Fetch data | `npm run fetch:daily` |
| Build prod | `npm run build` |
| Custom symbols | `$env:INGEST_SYMBOLS_FILE="path"` |
| TLS bypass | `$env:ALLOW_INSECURE_SSL="true"` |
